# Repurposing Engine — 1 Idea → Multiple Posts

## The Pipeline

One topic enters. Multiple platform-native outputs exit. Each output RETHINKS the topic for its format — it does NOT reformat the same text.

Reference: [[platforms/x]] for X rules. [[engine/content-types]] for formula selection.

## Repurposing Chain for X

Since @findocere is X-primary, the repurpose chain produces multiple X post VARIANTS from one idea:

### Input: One topic/idea/data point

### Output Options (pick 2-3 per topic):

1. **Thread version** (F3 or F7) — deep, numbered, screenshot-heavy
2. **Single tweet version** (F4 or F2) — compressed to 280 chars, one sharp insight
3. **Confession version** (F1) — emotional angle on the same data
4. **Quote-tweet-of-self version** — 3 days later, quote your own thread with an update
5. **Reply-thread version** — post the insight as a thoughtful reply to a relevant account's post

### Repurpose Timing

- Day 0: Main post (thread or single tweet)
- Day 1: Self-reply with additional insight or update
- Day 3: Quote-tweet own post with new angle or result update
- Day 7: Repackage as a different formula (e.g., thread → hack drop)
- Day 30: Compile top 5 into a pinned "journal" thread

## Cross-Pillar Repurposing

## Anti-Patterns

- NEVER copy-paste the same text across formats — each version must think about the topic differently
- NEVER repurpose a post the same day — let the original breathe
- NEVER repurpose a post that flopped without changing the hook — the hook was the problem
