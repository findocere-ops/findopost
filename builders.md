# Audience: Builders (AI x Crypto Crossover)

## Who They Are

- Developers building autonomous agents, bots, or tools for DeFi
- Familiar with LLM frameworks (Hermes, LangChain, CrewAI, etc.)
- Building on Solana or experimenting with on-chain AI integrations
- Care about architecture decisions, not just results
- Follow NousResearch, AI agent accounts, Solana dev accounts

## What They Care About

- Architecture patterns: how to structure agent → DeFi interactions safely
- Infrastructure pain: RPC latency, state caching, API reliability
- Safety constraints: why the LLM should NOT make trading decisions
- Real build logs: what actually works vs. what looks good in a demo
- Framework comparisons: Hermes vs. alternatives

## What Language Lands

DO:
- Technical specifics: "switched from live RPC calls to 30-second cached state"
- Honest failures: "the agent made a decision it wasn't supposed to — here's what broke"
- Architecture decisions: "the LLM suggests parameters, human approves — here's why"
- Tool mentions: specific frameworks, libraries, APIs

DON'T:
- Marketing language about AI capabilities
- Vague "AI will revolutionize DeFi" takes
- Oversimplified explanations that insult their intelligence
- Claims about agent performance without showing methodology

## How They Engage

- Reply with their own architecture decisions or pain points
- Quote tweet to add their perspective on agent design
- DM to collaborate or ask technical questions
- Bookmark implementation details

## Content That Converts This Audience

- Builder logs with specific technical lessons (F5)
- Architecture diagrams showing agent pipeline
- Side-by-side: agent suggestion vs. human decision
- Honest failure reports from agent testing
